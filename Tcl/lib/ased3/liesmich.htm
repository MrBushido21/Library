Installation:

Stellen Sie sicher, dass Tcl 8.0 oder höher installiert ist.

Windows:
Entpacken Sie die zip-Datei in ein Verzichnis Ihrer Wahl. Beachten Sie, dass die Option zum Wiederherstellen der Unterverzeichnisse in Ihrem Packprogramm aktiviert ist. Hierzu eignet sich z.B. Winzip oder Power Archiever 2000. Um ASED zu starten klicken Sie im Explorer doppelt auf die Datei ased.tcl.

Linux (Unix):
Entpacken Sie die *tar.gz-Datei in ein Verzeichnis Ihrer Wahl. Für dieses Verzeichnis benötigen Sie Schreibrechte. Zum Starten von ASED führen Sie die Datei ased aus.

Nach dem Entpacken sollte das Installationsverzeichnis (z.B. ~ASED) folgende Unterverzeichnisse haben:
~/ASED/BWidget-1.2.1
~/ASED/images
~/ASED/lang

Beim ersten Starten von ASED wird die Datei ased.cfg in Ihrem Home-Verzeichnis angelegt. Hier werden die von Ihnen gewählten Voreinstellungen gespeichert.

Tcl-Scripte testen
Wenn Sie den Test-Schalter der Werkzeugleiste anklicken, wird der Inhalt des aktuellen Fensters ausgeführt (Nicht der geöffneten Datei!!). Das erlaubt Ihnen Änderungen an einer Datei durchzuführen und direkt zu testen, ohne die Datei selbst sofort verändern zu müssen.
Wird eine Datei gespeichert, wird eine entsprechende Sicherheitskopie der Originaldatei angelegt. Sicherheitskopien erkennen Sie an einer angehängten Tilde ~. Die Sicherheitskopie der Datei test.tcl heißt demnach test.tcl~. Damit haben Sie immer die Möglichkeit die Original-Datei wiederherzustellen.
ASED bietet grundsätzlich zwei Möglichkeiten, Scripts zu testen. Sofern die Option Benutze eval-Server nicht aktiviert ist, startet ASED lediglich einen neuen Interpreter und führt in dessen Kontext das Script aus. Sollte das Script Fehler enthalten, die zum Aufhängen des Programms führen, lässt sich auch ASED nicht mehr bedienen. Damit sind auch eventuell noch nicht abgespeicherte Arbeiten verloren. Um dies zu vermeiden, kann die Option Benutze eval-Server aktiviert werden. In diesem Fall startet ASED zum Testen des Scripts ein Server-Script als eigenständige wish-Instanz. Dieser Server läuft völlig unabhängig von ASED. Die zu testenden Scripts werden nun zur Ausführung an diesen Server übergeben. Sollte hierbei ein Fehler auftreten, kann ASED i.d.R. weiterbedient werden, auch wenn das Testprogramm sich aufgehängt hat. In diesem Fall muß dann der Server vom Betriebssystem aus beendet werden. Normalerweise kann das getestete Programm jedoch über den Stop-Schalter der Werkzeugleiste beendet werden. Hierdurch wird der jeweilige Slave-Interpreter, der zur Ausführung des Scripts gestartet wurde beendet.
Darüber hinaus bietet diese Server-Option die Möglichkeit, die Tcl bzw. wish-Version mit der der Server und damit das zu testende Programm laufen soll auszuwählen. Somit ist es sehr einfach möglich, die Kompatibilität mit unterschiedlichen Tcl-Versionen zu testen oder auch Programme für Tcl-Varianten wie iTcl oder TclX zu entwickeln und zu testen. Voraussetzung hierfür ist natürlich, dass die jeweiligen Tcl-Versionen installiert sind.

Um eine Anwendung, die aus mehreren Dateien besteht zu testen, ist es nicht nötig, alle Dateien in den Editor zu laden. Sie können eine Startdatei festlegen, die jedes Mal ausgeführt wird, wenn Sie den Test-Schalter anklicken. Wenn nötig können Sie für unterschiedliche Dateien auch unterschiedliche Startdateien angeben. Auch in diesen Fällen wird der Inhalt des aktuellen Editor-Fensters und NICHT die entsprechende Original-Datei ausgeführt, sodaß auch hier bequem schnell Änderungen ausgetestet werden können, ohne jedes Mal die Änderungen vorher abspeichern zu müssen.

Wenn Sie Tcl-Scripte über den Server ausführen erscheint im Konsolen-Fenster eine zusätzliche Anzeige. Hier werden alle Ausgaben des getesteten Programms angezeigt. Zusätzlich ist es möglich direkt im Kontext der getesteten Anwendung Tcl-Kommandos auszuführen. So ist es möglich, während der Ausführung eines Programms z.B. Variablen-Werte abzufragen oder auch zu setzen.  Grundsätzlich können alle zum jeweiligen Zeitpunkt gültigen Befehle (also auch die procs des Programms) zur Laufzeit aufgerufen werden.

Einer zu testenden Anwendung können zusätzliche Argumente übergeben werden. Diese können einfach in die rechte Kombobox der Werkzeugleiste eingetragen werden.

Konsole
Unter Windows gibt es typischerweise kein Terminal-Fenster. Deshalb stellt ASED ein eigenes Konsolenfenster bereit. Die Ausgaben des puts-Kommandos werden in dieses Konsolenfenster umleitet. Somit kann auch unter Windows ein einfaches Debuggen erfolgen, indem puts-Kommandos in den Code eingebaut werden. Dies funktioniert derzeit jedoch noch nicht im Zusammenhang mit Slave-Interpretern.

Sourcecode Navigation
ASED stellt folgende Möglichkeiten für ein schnelles Navigieren im Sourcecode zur Verfügung

·        Code Browser Fenster
Dieses Fenster zeigt alle procs, namespaces, classes und methods in einer Baumstruktur an. Wenn Sie einen entsprechenden Eintag anklicken, wird im Editorfenster automatisch die entsprechende Codezeile angesteuert.

·        History Buttons (im unteren Teil des Code Browser Fensters)
ASED merkt sich die letzten Positionen im Code die von Ihnen editiert wurden. Mit den History Buttons können Sie schnell zwischen den letzten bearbeiteten Codezeilen hin- und herspringen. Allerdings wird innerhalb der gleichen proc immer nur eine Position gespeichert.

·        Gehe zu Zeile- Kombobox (im unteren Teil des Code Browser Fensters)
Diese Kombobox zeigt immer die aktuelle Zeile an. Wenn Sie den Schalter rechts daneben drücken, wird die angezeigt Zeile in der Zeilenhistorie gespeichert. Sie können auch selbst eine Zeilennummer eingeben und somit zu der angegeben Zeile springen. Es ist auch möglich relative Sprünge einzugeben. Dies ist ganz nützlich, wenn Ihnen Tcl eine Fehlermeldung bringt. Normalerweise wird dann dort die entsprechende Procedur und die Zeile angegeben. Sie können mit ASED im Code Browser die entsprechende Proc anspringen und geben dann in der Zeilen-Kombobox die Zeile relativ an, also z.B. +10.

·        Quicksearch (Schnelle Suche) über die Werkzeugleiste
Um in einer geöffneten Datei zu suchen brauchen Sie nur den entsprechenden Text in die Such-Kombobox der Werkzeugleiste einzugeben. Drücken von RETURN startet die Suche vorwärts. Mit den Pfeilen rechts daneben können Sie entsprechend vorwärts oder rückwärts suchen. Für eine erweiterte Suche können Sie den Suchen-Schalter der Werkzeugleiste benutzen. Dieser ruft einen Suchen-Dialog auf, in dem Sie erweiterte Einstellungen vornehmen können.

·        In Dateien suchen
Wenn Sie nicht mehr genau wissen, in welcher Datei die von Ihnen gesuchte proc oder Funktion steckt hilft Ihnen der Menüpunkt In Dateien suchen im Menü Bearbeiten weiter. Beschreibung s.u. Grafische Oberfläche für "grep"

Kontext Menüs

Wenn Sie im Editor oder im Code Browser mit der rechten Maustaste klicken erscheint ein Popup-Menü.

Wenn Sie im Editor die Funktion Markierung automatisch einrücken auswählen ohne etwas markiert zu haben, wird diese Funktion für den gesamten Code der aktuell geöffneten Datei ausgeführt. Hierbei werden alle tabs durch vier Leerzeichen ersetzt.

Hervorheben von zusammengehörenden Klammern

Um bei einer Klammer (normal, geschweift oder eckig) die dazugehörige Klammer angezeigt zu bekommen müssen Sie links von der entsprechenden Klammer mit der Maus klicken.

Grafische Oberfläche für "grep"

Wenn Sie die Funktion In Dateien suchen aus dem Menü Bearbeiten wählen, öffnet sich ein Dialogfenster, in dem Sie Optionen setzen können, um komplette Verzeichnisse nach verschiedenen Kriterien zu durchsuchen. Die Ergebnisse werden in einem Ergebnisfenster angezeigt. Ein einzelner Klick auf einen Eintrag öffnet die entsprechende Datei im Editor und springt automatisch zu diesem Eintrag.

"watch cursor" zurücksetzen

Wenn während des Testens von Tcl-Scripten ein Fehler auftritt, bleibt der Cursor manchmal als Sanduhr weiter sichtbar. Um den Cursor wieder auf die Pfeilanzeige zurückzusetzen, müssen Sie (mit der Sanduhr) auf den Stop-Schalter der Werkzeugleiste klicken.

evalClient und evalServer

evalClient.tcl und evalServer.tcl können auch als eigenständige Anwendungen laufen und stellen so ein Beispiel für eine Client-/Server-Anwendung dar. Wenn Sie evalClient.tcl starten wird automatisch auch evalServer.tcl gestartet. Der Client stellt ein einfaches Terminal-Fenster zur Verfügung mit dem über den Server Tcl-Kommandos ausgeführt werden können oder auch komplette Anwendungen. Wenn Sie Anwendungen über das Terminal starten, sollten Sie die Ausgaben des Programms im Terminalfenster sehen. Außerdem werden alle weiteren Kommandos, die Sie eingeben, im Kontext der Anwendung ausgeführt (solange die Anwendung läuft).

Hotkeys

Strg-c: kopieren

Strg -x: ausschneiden

Strg -v: einfügen

Strg -y: aktuelle Zeile löschen

Strg -l: letzte Suche wiederholen (arbeitet nicht mit der Schnell-Suche Kombobox, sondern nur im Zusammenspiel mit dem Such-Dialog)

Anmerkung: Wenn die Eingabezeile einer Kombobox den Eingabefokus hat, brauchen Sie nur die Return- bzw. Eingabe-Taste zu drücken um die Funktion auszulösen (z.B. für Schnellsuche)
